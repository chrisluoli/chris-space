Claude Skill-Creator

Anthropic 官方 · 元技能（用 Skill 创建 Skill）
Meta-Skill 官方标准 CLI 模式
Anthropic 官方推出的"用 AI 创建 AI 技能"的元技能框架。不仅是一个模板生成器，而是包含创建→评测→优化完整闭环的工程化系统。
  • 🔥对话式创建 — 自然语言描述需求，自动生成 SKILL.md
  • 🧪多智能体评测 — Grader（评分）+ Comparator（盲测）+ Analyzer（分析）
  • 🔄描述优化循环 — Train/Test Split + 盲历史防过拟合
  • 👁️Eval Viewer — 交互式人工审核界面
  • 📐渐进式加载 — 元数据→正文→资源 三层按需加载
💡 最大亮点：将 Skill 开发建模为机器学习问题——有训练集/测试集、有评分器、有盲测对比、有迭代优化。不是一次性生成，而是持续进化。
⭐⭐⭐⭐⭐ 工程化程度最高 · 企业级首选
⚡

Superpowers

obra · 复杂任务规划与拆解引擎（27K+ ⭐）
任务拆解 需求澄清 结构化输出
专注解决"从模糊想法到可执行计划"的难题。通过发散→收敛→拆解三阶段，将复杂需求转化为原子化操作序列。
  • 🧠三阶段处理 — 发散（头脑风暴）→ 收敛（PRD）→ 拆解（原子操作）
  • ⏱️原子化拆解 — 每个任务单元 2-5 分钟可完成
  • 📋风险检查清单 — 自动识别潜在风险和前置依赖
  • 🎯防幻觉设计 — 强制分步确认，防止 Agent 半途而废
💡 最大亮点：将需求拆解本身做成了一套完整方法论。"发散-收敛-拆解"三阶段模型可以直接借鉴到 Skill 创作的需求澄清环节。
⭐⭐⭐⭐⭐ 27K Stars · 需求处理最强
🤖

Coze 扣子

字节跳动 · 新一代 AI 应用开发平台
零代码 工作流 多渠道发布
面向非技术用户的 Bot 创建平台。提供对话式创建 + 手动创建两种模式，支持工作流可视化编排和知识库接入。
  • 💬双模式创建 — 对话式（AI 引导）或 手动填表创建
  • 🔧可视化工作流 — 拖拽式节点编排，开始→处理→结束
  • 📚知识库接入 — 上传 PDF/Doc/Excel，自动建立检索
  • 📱多渠道发布 — 飞书、微信、网页、API 一键发布
  • 🔌插件生态 — 600+ 内置插件，搜索/绘图/代码运行等
💡 最大亮点：极低的使用门槛 + 强大的工作流编排。「提示词 + 工作流 + 知识库 + 插件」四要素组合的设计思路值得参考。
⭐⭐⭐⭐ 用户量最大 · 小白友好度最高
✨

GPT Builder (Custom GPTs)

OpenAI · 自定义 GPT 创建工具
对话创建 GPT Store 知识上传
业界首个"用对话创建 AI Agent"的产品。通过自然语言描述就能创建自定义 GPT，开创了对话式创建范式。
  • 🗣️纯对话式 — "You are an expert..." 即可创建
  • 📎知识上传 — 上传文件作为 GPT 的知识源
  • 🔧Actions — 接入外部 API（OpenAPI spec）
  • 🏪GPT Store — 分享和发现社区创建的 GPTs
  • ⚙️配置面板 — 对话创建 + 手动配置双视图
💡 最大亮点：开创了「对话即创建」的交互范式。「Create + Configure」双面板设计成为后续产品的标准参考。
⭐⭐⭐⭐ 首创者 · 品牌影响力最大
🔮

Dify

开源 · LLM 应用开发平台
开源 Workflow RAG Agent
功能最全面的开源 LLM 应用开发平台。提供可视化 Workflow、RAG 引擎、Agent 编排和 Prompt IDE 等完整工具链。
  • 🎨Prompt IDE — 可视化编辑 + 变量 + 模板 + 实时调试
  • 🔀Workflow 编排 — DAG 有向图，支持条件分支和循环
  • 🗃️知识管理 — 向量检索 + 多种分块策略
  • 📊可观测性 — 日志、Token 消耗、执行链路追踪
  • 🧪批量测试 — Dataset + 批量运行 + 结果对比
💡 最大亮点：Prompt IDE + 批量测试的组合。可以在编辑器内直接调试 Prompt 效果，用数据集批量验证质量——这是我们的「质量引擎」可以直接借鉴的。
⭐⭐⭐⭐⭐ 开源领域功能最全
🚢

SkillPort

gotalab · 跨平台 Skills 管理中枢
跨平台 版本管理 格式转换
解决 Skills 分散管理的痛点。统一管理 Claude Code、Codex、Copilot 等不同 Agent 平台的技能，支持批量导入和格式转换。
  • 📦统一管理 — GitHub 仓库 + 本地文件批量导入
  • 🔄自动转换 — 不同 Agent 格式互转
  • 📌版本追踪 — Git 集成 + 自动更新检测
  • 🔌多平台适配 — Claude/Codex/Copilot/VS Code
💡 最大亮点：把 Skills 当作"包管理"来做。版本控制 + 跨平台兼容 + 集中管理的思路可以作为 Phase 3 的参考方向。
⭐⭐⭐ 管理层面最完善
🌊

FlowGPT

社区驱动 · Prompt 分享与发现平台
社区 Prompt市场 排行榜
最大的 Prompt 社区平台。用户可以分享、发现和 Fork 他人的 Prompt，通过社区投票和使用数据形成排行榜。
  • 🏪Prompt 市场 — 按场景/类型分类浏览
  • 🔀Fork 机制 — 一键复制并修改他人 Prompt
  • 📈排行榜 — 按使用量/评分/热度排序
  • 💬在线测试 — 直接在平台上运行 Prompt
💡 最大亮点：社区生态 + Fork 机制。「不从零开始，而是站在他人肩膀上」的理念可以应用到 Skill 模板库的设计中。
⭐⭐⭐ 社区最活跃
🧬

Agent-Skills Context Engineering

muratcankoylan · 上下文工程方案（6K+ ⭐）
上下文优化 多智能体 记忆系统
专注解决长上下文场景下的 Token 浪费和"中间丢失"问题。提供多智能体分布式处理和跨会话记忆系统。
  • 🧠记忆系统 — 跨会话持久化存储与检索
  • 🔀多 Agent 协作 — 分布式任务降低单 Agent 负载
  • 📊性能评估 — 可量化的 Agent 性能基准测试
  • ⚡Token 优化 — 智能裁剪和上下文压缩
💡 最大亮点：渐进式披露 + Token 经济学。把"SKILL.md 主文件 ≤500行" + 按需加载 REFERENCE.md 的架构设计做得很深入。
⭐⭐⭐⭐ 上下文管理最深入

从竞品中提炼的 8 条核心设计原则
🎯

渐进式加载架构

Skill 内容分三层：元数据→正文→资源，按需加载。主文件 ≤500 行，详情放 REFERENCE.md。解决 Token 瓶颈。

— 来自 Claude Skill-Creator & Context Engineering
⚖️

自由度分级控制

根据任务风险分三级：低自由度（数据库操作→精确脚本）、中自由度（常规→伪代码）、高自由度（创意→方向指引）。

— 来自 Claude Skills 7大工程原则
🧪

ML 化质量评测

把 Skill 质量建模为 ML 问题：有训练集/测试集，有盲测对比（Comparator Agent），有防过拟合（Blinded History）。

— 来自 Claude Skill-Creator 评测模块
💬

对话 + 配置双面板

GPT Builder 开创的「Create（对话创建）+ Configure（手动配置）」双视图已成行业标准。满足新手和高级用户。

— 来自 GPT Builder & Coze
🔀

Fork + 社区生态

不从零开始创建，而是Fork 高质量模板→微调→发布。社区排行榜、使用数据驱动的推荐。

— 来自 FlowGPT & GPT Store & Coze Bot Store
🧠

发散-收敛-拆解模型

处理模糊需求的三阶段：发散（头脑风暴所有可能）→收敛（聚焦为 PRD）→拆解（原子化为 2-5 分钟任务）。

— 来自 Superpowers（27K ⭐）
📐

解释 Why，不强制 Must

向模型解释任务本质和逻辑，而非死记硬背步骤。少用 MUST/NEVER，多解释原因。让 AI 理解而非记忆。

— 来自 Claude Skills 设计哲学
🔄

Claude A/B 迭代开发

Claude A（架构师）写 Skill → Claude B（测试员）执行 → 观察失败 → 反馈修正 → 再测试。形成自动化闭环。

— 来自 Claude Skills 实战指南

📋 具体改进方案
# 改进点 参考来源 实现建议 优先级
1 需求处理升级为「发散-收敛-拆解」 Superpowers 阶段1增加发散式头脑风暴，AI先列出所有可能的功能点，再引导用户聚焦 P0
2 增加 A/B 对比测试 Claude Skill-Creator 质量验证阶段，展示「有Skill vs 无Skill」的效果对比 P0
3 增加「Create + Configure」双面板 GPT Builder 左侧对话创建 + 右侧可直接编辑配置项，两者实时同步 P0
4 自由度分级 UI Claude Skills 原则 每个 Step 允许用户设置「约束等级」：严格/标准/灵活 P1
5 模板 Fork 机制 FlowGPT + GPT Store 模板库支持 Fork → 自定义 → 发布，形成社区沉淀 P1
6 批量测试数据集 Dify 上传测试用例 CSV，批量运行 Skill 并对比结果 P1
7 可视化工作流预览 Coze + Dify 在逻辑流程图 tab 中增加可拖拽节点编排能力 P2
8 跨平台 Skill 导出 SkillPort 支持导出为 Claude Skill / Cursor Rules / Copilot Instructions P2
Anthropic 的核心洞察："Skills 是 Agent 的「大脑回路」（经验与流程），MCP 是 Agent 的「手脚」（工具与连接）。两者互补，不是替代。"

这意味着我们的产品不应只关注 Skill 文档本身，还应考虑与 MCP 工具的协同——在 Skill 创作过程中自动推荐相关的 MCP 工具接入。
— Claude Agent Skills 实战指南
Superpowers 的方法论："将非确定性的模糊需求转化为确定性的执行路径。每个任务单元 2-5 分钟可完成。"

这提示我们：阶段1的需求澄清不应该是简单的问答，而应该是结构化的需求工程——先发散所有可能性，再收敛到可执行范围。
— Superpowers README (27K ⭐)
