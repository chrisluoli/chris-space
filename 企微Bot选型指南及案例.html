💡 一句话特征总结
🤖 智能机器人：能被拉进群、能读文件、能流式输出打字、不能主动发文件消息，适合拟人化数字员工场景
📢 群机器人：能被拉进群、能发文件消息、不能读文件消息、不能流式输出，适合群管理、群运营场景
🎧 客服号：能对接坐席系统、能主动发消息给个人用户、能自定义展示门户、不支持流式输出，适合人工客服运营场景

⚖️ 核心优劣势总结

🤖 智能机器人

✦ 优势
  • 唯一支持流式响应，长回答体验极佳
  • 支持思考折叠，展示推理过程
  • 富媒体消息类型齐全
  • 支持引用消息，上下文清晰
✦ 劣势
  • 主动推消息有限制
  • 不能管理群聊

📢 群机器人

✦ 优势
  • 免审批，自助创建，门槛最低
  • 可自助管理回调，开发最灵活
  • 支持主动发送群消息
  • 可以被客服号 @触发联动
✦ 劣势
  • 不支持流式响应
  • 单条消息限4K字符
  • 不支持引用、文件类消息
  • 个人推送需申请权限

🎧 客服号

✦ 优势
  • 功能最全面（群管理+主动推送+监听）
  • 支持主动发消息（群/个人都行）
  • 可用接口管理群聊
  • 全公司可见，触达面最广
✦ 劣势
  • 不支持流式响应
  • 单条消息限4K字符

⚡ 合规 Skill 建设思路

创造一份 Skill，业务最重要的事情是干什么？

1
明确 Skill 需求

合规 Case 种类多，处理思路各异：

孕期解除 单方解除 聚众闹事 ...

先定优先级：当前单方解除最高频，优先复刻。

2
提炼 Case 分析逻辑
⭐ 重中之重

还原合规同学真实处理流程：

❓ 查哪些法律条款？
❓ 了解员工哪些情况？
❓ 是否报备法律部门？
❓ 是否需要填写表格？

💡 业务输出核心处理流程，是 Skill 的灵魂。

🚩 合规业务同学正在重点打磨此步骤
📄 ER合规助手 Skill 逻辑示例 → 📄 单方解除 Skill 主流程 →
3
Skill 工程与开发

需产研协同落地：

🔧 外部知识接口对接
🔧 用户语义动态参数解析
🔧 内外部知识冲突处理
🔧 先解析后执行流程设计
4
上线验证

开放真实用户使用，收集反馈、持续迭代。
